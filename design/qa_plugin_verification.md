## QA 핵심 추가 검증 — 회귀 테스트 fail-closed (백팀장 미수행)

"10/10 통과"만으론 테스트가 진짜 검사하는지 모른다. 결함을 심어 RED가 되는지 확인(원본 불변, 임시복제 개악):

| 주입 결함 | 기대 | 결과 |
|-----------|------|------|
| `remote_exec` 패턴 삭제(D-Q2 방어선 제거) | 미탐 테스트 RED | ✅ `test_dq2_curl_space_sudo_sh` FAIL → exit 1 |
| `subprocess` 패턴을 bare로 되돌림(D-Q1 회귀) | 오탐 테스트 RED | ✅ `test_dq1_import_subprocess_not_flagged` FAIL → exit 1 |
| 스캐너 무력화(--strict도 exit0) | 위험 테스트 RED | ✅ `test_dq2_curl_pipe_sh` FAIL → exit 1 |

**→ 회귀 스위트가 진짜 fail-closed. 방어선을 건드리면 반드시 RED.** 내가 요구한 "고정 케이스 등록"이 형식이 아니라 실효를 갖췄음을 확인.

## QA 판정
- **백팀장 스켈레톤: QA 재현 통과.** 개별 컴포넌트 전부 실측 확인 + 회귀 테스트 실효성까지 검증.
- **QA 조건(D-Q1/D-Q2 회귀 고정) 이행 확정.** 데이터 분리(lib/)·opt-in 차단(--strict)도 내 설계 §2 반영됨.
- **미해결(백팀장 보고와 일치)**: `claude --plugin-dir` 런타임 로드는 claude CLI 미인증으로 보류. 이건 코드 결함이 아니라 환경(인증) 이슈 → 수장 확인 대기. 개별 컴포넌트가 다 통과했으므로 통합 로드는 인증만 풀리면 확인 가능.
- **잔여 QA 관찰**: 훅의 BLOCK_PATTERNS(런타임 명령 차단)와 스캐너 danger_patterns(생성코드 검사)가 **별도 목록**이라 이중관리 위험. Phase 2에서 가능한 한 단일 소스로 수렴 권고(지금은 용도가 달라 분리 타당).

## [갱신 2026-07-05] 백팀장 이중관리 조사 + 훅 fail-closed — QA 재현

백팀장이 이중관리 조사 중 "측정 함정" 발견·조치 + 훅 회귀 신설(18/18) → QA 직접 재현.

### QA 재현 결과 (전부 확정)
- **전체 회귀 18/18**: pytest 경로 `18 passed`, 폴백 경로 스캐너 10/10 + 훅 8/8 ✅
- **훅 fail-closed 실효 확인**(백팀장 미명시 부분까지): 훅 방어패턴 제거 시 RED 확인 —
  - `rm -rf` 방어 제거 → `test_block_rm_rf_root` FAIL(exit 1) ✅
  - `curl|sh` 방어 제거 → `test_block_curl_pipe_sh` FAIL(exit 1) ✅
- **gap_probe.sh 제외 확인**: `.sh`라 pytest 미수집 + test_guard_hook.py는 docstring에서 언급만(실행 참조 0). "회귀에서 제외" 실제 성립. (grep -l 오탐은 주석 매칭이었음 — 정독으로 확정)

### 측정 함정 = QA가 반드시 새길 교훈
- 백팀장의 gap_probe 오측 원인(위험명령 셸 조립 → 환경가드가 exit code 오염 → 전부 pass 오측)은 **정확한 진단**. QA가 지난 턴 `rm -rf /`를 훅에 넣을 때 터미널 안전가드 승인대기에 걸린 것과 동일 근원.
- **QA 표준 채택**: 훅/스캐너 테스트는 위험문자열을 명령줄에 노출 금지, JSON을 tempfile→stdin 주입 후 exit code만 판정. (test_guard_hook.py 방식이 정답. 메모리에도 기록.)
- 소견: "측정 도구 자체가 거짓말할 수 있다"는 것은 QA의 핵심 경계 대상. 백팀장이 오보 직전 스스로 잡은 것은 good catch. 다만 신뢰불가 측정기는 삭제보다 **교훈 주석+회귀 제외**로 남긴 판단에 동의(반면교사 보존).

### QA 판정
- **훅+스캐너 회귀 스위트: 재현 통과.** 양쪽 다 fail-closed 실효 확인.
- **이중관리**: Phase 2 단일소스 수렴 항목으로 확정 등록. 지금 분리는 용도차(런타임 셸 vs 생성 diff)로 타당.
- **미해결**: claude CLI 인증(런타임 통합 로드) — 코드 아닌 환경 이슈, 수장 확인 대기(변동 없음).
