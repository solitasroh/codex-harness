# ADR-0001: envlint는 Python3 표준 라이브러리만으로 구현한다

- 상태: 승인
- 날짜: 2026-07-05
- 관련: docs/spec/envlint.spec.md 확정 스펙

## 컨텍스트
`.env` 검증기를 만든다. 배포·CI·git 훅에 물릴 작은 도구라 설치 부담이 없어야 하고,
어느 환경에서든 바로 돌아야 한다. 외부 패키지 의존은 이식성·설치 마찰을 늘린다.

## 결정
우리는 envlint를 Python3 표준 라이브러리(argparse, sys, json, re)만으로 구현한다.
외부 패키지(python-dotenv 등)에 의존하지 않는다.

## 대안
- python-dotenv 의존 — .env 파싱을 검증된 라이브러리에 위임. 버린 이유: 설치 부담,
  그리고 dotenv의 파싱 관용이 우리가 잡으려는 "형식 오류"를 조용히 흡수해버려 검증 목적과 충돌.
- Node/Go로 단일 바이너리 — 배포는 좋으나 이 프로젝트 관행(Python3)과 어긋나고 빌드 추가.

## 결과
- 긍정: 설치 0, 이식성 최대, 프로젝트 관행 일치, 테스트 단순.
- 부정/비용: .env 파싱을 직접 구현해야 함(따옴표·export 접두 등 엣지 직접 처리).
- 후속: 파싱 로직에 엣지케이스 테스트를 반드시 붙인다(스펙 엣지케이스 항목 전부 커버).
