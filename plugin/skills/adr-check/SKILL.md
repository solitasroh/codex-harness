# adr-check: 설계 결정 기록 & 충돌 감지

design-grill이 요구사항을 확정하면, 이 스킬이 **결정을 문서로 못 박고** 기존 결정과의
충돌을 잡는다. 목적: 결정 드리프트(문서 따로 코드 따로) 차단. 자기보고 말고 **파일을 실제로 읽어** 대조한다.

## 0. 대상 파일 (없으면 생성)
- ADR: `docs/adr/NNNN-<slug>.md` (4자리 순번, 예 `0007-use-mcp-for-codex.md`)
- 용어집/컨텍스트: `CONTEXT.md`(있으면). 도메인 용어·제약·기존 원칙의 단일 소스.

## 1. ADR 기록 포맷 (그대로 채운다)
```markdown
# ADR-NNNN: <결정을 한 문장으로>

- 상태: 제안 | 승인 | 폐기 | 대체됨(→ ADR-MMMM)
- 날짜: YYYY-MM-DD
- 관련: design-grill 확정 스펙 <링크/요약>

## 컨텍스트
왜 이 결정이 필요한가. 배경·제약·강제 요인(기술/조직/시간).

## 결정
우리가 택한 것. 능동태 단정형("우리는 X를 쓴다"). 모호어 금지.

## 대안
검토했으나 버린 선택지 + 버린 이유(각 1~2줄). 최소 1개.

## 결과
- 긍정: 이 결정으로 좋아지는 것.
- 부정/비용: 감수하는 트레이드오프, 새로 생기는 제약.
- 후속: 이 결정이 강제하는 다음 작업/재검토 조건.
```
규칙: **대안·결과 비우지 않는다.** 대안 없는 결정은 "검토 안 함"과 같아 반려 대상.

## 2. 충돌 감지 로직 (파일 실제 대조)
1. `docs/adr/` 전체를 읽어 **상태=승인**인 ADR의 "결정" 문장을 수집.
2. 이번 새 결정과 대조. 충돌 신호:
   - 같은 주제(같은 컴포넌트/키워드)에 **상반된 선택**(예: 기존 "REST 사용" vs 신규 "gRPC 사용").
   - `CONTEXT.md` 용어집의 확정 용어·제약을 위반(예: 용어집이 "인증=OAuth"인데 신규가 API키 하드코딩).
3. **충돌 발견 시 자동 진행 금지.** 사용자에게 알린다:
   - 어떤 ADR(번호·제목)과, 어느 지점이, 어떻게 충돌하는지 구체적으로.
   - 선택지 제시: (a) 새 결정 철회 (b) 기존 ADR을 "대체됨"으로 표시하고 신규 승인 (c) 재설계.
4. 충돌 없으면 새 ADR 상태를 "승인"으로 기록.

## 3. 인라인 갱신 (드리프트 차단)
결정이 확정(승인)되면 관련 문서를 **그 자리에서** 갱신한다. 나중으로 미루지 않는다.
- `CONTEXT.md` 용어집에 새 용어·제약 추가.
- 기존 ADR을 대체하면 그 ADR 상태를 `대체됨(→ ADR-NNNN)`으로 수정.
- 결정이 코드 규약을 바꾸면 관련 README/규약 문서에 한 줄 반영.
- 갱신한 파일 목록을 사용자에게 보고(무엇을 어디서 바꿨는지).

## 4. 산출 → 다음 단계
ADR이 승인·기록되면 확정 스펙 + ADR 번호를 **Codex 코딩 위임(harness-run 3단계)**로 넘긴다.
충돌 미해소 상태로는 코딩 단계로 절대 넘어가지 않는다(게이트).
